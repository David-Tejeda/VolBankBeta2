Details:
Se debe de tener registro de la información básica del voluntario asi como sus habilidades y su dirección<br/>
Scenario Type
Escenario principal Basic Path
Details:
1. El voluntario se autentifica en el sistema<br/>2. El voluntario señala que realizará un nuevo deposito<br/>3. El voluntario ingresa la cantidad de tiempo que desea depositar (fecha de inicio/final)<br/>4. El voluntario describe brevemente cuales son sus capacidades<br/>5. El sistema toma la información y la guarda dentro de la lista de servicios<br/>6. El sistema le manda al servidor web el nuevo deposito<br/>7. El sistema le notifica al voluntario que el deposito ha realizado correctamente.<br/>
*a Un organizador realiza un deposito Alternate
Details:
1. El organizador se autentifica en el sistema<br/>2. El organizador señala que realizará un nuevo deposito<br/>3. El organizador ingresa la cantidad de tiempo que el voluntario desee depositar (fecha de inicio/final)<br/>4. El organizador describe brevemente cuales son sus capacidades del voluntario<br/>5. El sistema toma la información y la guarda dentro de la lista de servicios<br/>6. El sistema le manda al servidor web el nuevo deposito<br/>7. El sistema le notifica al organizador que el deposito ha realizado correctamente.<br/>8. El sistema envia un correo de notificación al correo del voluntario.<br/><br/>
